Problem & Solution

你的收藏夹，不再吃灰

每个开发者都遇到过这些问题。Starcat 用 AI + 本地优先的方式，逐一解决。

收藏了上千个仓库，想找一个却翻半天

Starcat 的解法

FTS5 全文搜索引擎

对仓库名、描述、Topics、笔记建立全文索引。支持自然语言查询和结构化组合过滤， 毫秒级返回结果。

三个月后点开一个 Star，不记得为什么收藏它

Starcat 的解法

AI 自动生成结构化摘要

AI 阅读 README 后给出中文摘要：项目做什么、解决什么问题、 技术栈是什么——你收藏时的"灵光一闪"不再丢失。

一个个手动打标签太累，不打又找不到

Starcat 的解法

AI 智能标签推荐

AI 分析仓库内容后推荐 3-8 个标签，带置信度评分。 支持 14 个预设分类体系，同义标签自动检测。你只需点确认。

Star 的库更新了，你永远是从别人的推文里知道的

Starcat 的解法

Release 订阅追踪 + 通知

订阅你关心的仓库，新版本发布时第一时间推送通知。 时间线视图统一展示，已读/未读一目了然。

全局搜索 & 结构化过滤

FTS5 全文搜索引擎对仓库名、描述、Topics、笔记建立索引，毫秒级返回结果。支持按语言、标签、状态组合过滤，字段级精准匹配。

AI 阅读 README 后生成结构化摘要：项目做什么、技术栈是什么、适用场景有哪些。摘要按仓库缓存，适合快速判断项目价值。

按需翻译 README HTML 并保留原始结构。缓存命中时直接展示，源 README 变化后提示重新翻译。

FAQ

下载前你可能想确认

Starcat 现在是 macOS 专属工具。这里把版本边界、数据存储和 AI Key 说清楚。

Starcat 支持哪些系统？

当前仅支持 macOS 15 及以上版本，Direct 下载包面向 Apple Silicon Mac。暂不提供 iOS、iPadOS、watchOS、Windows 或 Android 版本。

Direct 版本和 App Store 版本有什么区别？

当前落地页默认提供 Direct 版本下载，并通过 Sparkle 支持应用内更新。App Store 渠道会按单独发布流程处理，下载区会在可用时明确标注。

我的数据存在哪里？

Starcat 采用本地优先设计。仓库缓存可重建，标签、笔记、状态等长期数据写入你 Mac 上的本地 SQLite。

AI Key 会不会被 Starcat 托管？

Starcat 采用 BYOK 思路，你可以配置自己的模型服务或 API Key。页面中涉及 AI 的能力以上线版本实际开放范围为准。

Pro 授权设备数怎么计算？

每个 Pro 方案可在你本人使用的最多 2 台 Mac 上激活。Pro Yearly 相比月付节省约 37%。Lifetime 为一次性买断，包含全部 Pro 功能，无需续费。

以后会支持 iPhone 或 iPad 吗？

短期重点是把 macOS 版本打磨稳定。未来平台计划会在更新记录中说明，当前页面不承诺多端版本。
